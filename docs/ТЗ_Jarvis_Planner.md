**Версия ТЗ:** 2.0  
**Дата:** 27.02.2025  
**Проект:** iOS/macOS приложение + Python бэкенд

---

## 1. Назначение продукта

**Jarvis Planner** — приложение для планирования задач, просмотра календаря и почты (Google), учёта здоровья (питание, сон, активность) и аналитики с советами на основе ИИ. Язык интерфейса — русский.

### 3.2. Экран «План» — `PlannerView.swift`

- **Заголовок:** AI Planner  
- **Фон:** JarvisTheme.background  
- **Toolbar:** кнопка синхронизации с календарём (иконка `calendar.badge.plus`).

**Блоки:**

1. **Популярные задачи**  
   Горизонтальный скролл чипов. По умолчанию: «Тренировка», «Прогулка», «Медитация», «Звонок», «Учёба», «Работа», «Чтение» + до 4 частых из истории. Тап по чипу подставляет текст в поле «Новая задача».

2. **Форма новой задачи**  
   - Текстовое поле «Новая задача»  
   - Кнопка микрофона (вкл/выкл запись, заполнение полей из распознанного текста или AI)  
   - Поле «Заметки»  
   - DatePicker «Когда»  
   - Кнопка «Добавить» (создаёт задачу и напоминание)  
   - Кнопка «AI» (разбор текста/голоса в задачу через выбранную AI-модель)

3. **Список задач**  
   Заголовок «Задачи». Если список пуст — текст «Пока нет задач».  
   Для каждой задачи: чекбокс (выполнено/не выполнено), название, дата/время, иконка будильника при наличии напоминания, кнопка «ещё» — открывает модальное окно переноса.

**Модальное окно «Отложить»** (`RescheduleSheet`): заголовок «Отложить», название задачи, DatePicker «Новое время», кнопки «Отмена» и «Сохранить». Сохранение обновляет задачу и перепланирует уведомление.

### 3.3. Экран «Календарь» — `CalendarView.swift`

- **Заголовок:** Календарь  
- **Фон:** JarvisTheme.background  

**Состояния (ZStack):**

1. **Проверка авторизации** (`checkingAuth`) — индикатор «Проверка...».  
2. **Не авторизован** — блок «Подключите Google»: иконка календаря, текст «Войдите в аккаунт Google, чтобы видеть события календаря и почту», кнопка «Войти через Google» (открывает `AuthService.shared.openAuthInBrowser()`).  
3. **Ошибка** — иконка предупреждения, текст ошибки, кнопка «Повторить» (повторный запрос событий).  
4. **Нет событий** — иконка «calendar.badge.exclamationmark», текст «Нет событий».  
5. **Список событий** — вертикальный скролл карточек. В каждой: название, дата/время, краткий превью заметок (до 80 символов, замена `<br>` на пробел).

**Toolbar:** при успешной авторизации и отсутствии ошибки — кнопка обновления (иконка `arrow.clockwise`).  
При появлении экрана: `checkAuth()`. Pull-to-refresh: снова `checkAuth()`, при успехе — загрузка событий.

**Данные:** `CalendarService.shared.fetchEventsAsDTO()` → маппинг в `CalendarEventItem` (id, title, notes, startDate).

---

### 3.4. Экран «Почта» — `MailView.swift`

- **Заголовок:** Почта  
- **Фон:** JarvisTheme.background  

**Состояния:** те же по смыслу, что у Календаря:

1. Проверка авторизации.  
2. Не авторизован — «Подключите Google», «Войдите в аккаунт Google, чтобы видеть письма», кнопка «Войти через Google».  
3. Ошибка — текст ошибки, «Повторить» (повторная загрузка почты).  
4. Нет писем — «Нет писем».  
5. Список писем — карточки: тема (или «(Без темы)»), отправитель, сниппет (до 2 строк).

**Toolbar:** при авторизации и без ошибки — кнопка обновления.  
Загрузка: `MailService.shared.fetchMessages(maxResults: 15)`. Ошибка `MailError.notAuthorized` обрабатывается отдельно (сообщение «Требуется авторизация» или из ответа).

---

### 3.5. Экран «Аналитика» — `AnalyticsView.swift`

- **Заголовок:** Аналитика  
- **Фон:** JarvisTheme.background  

**Блоки:**

1. **Статистика**  
   По дням (дата, всего задач, выполнено). Если данных нет — «Нет данных для анализа».

2. **Совет**  
   Один текст в зависимости от доли выполненных задач по всем задачам:  
   - ≥80% — «Отличный баланс! Продолжай в том же духе.»  
   - 40%–80% — «Хороший темп. Попробуй дробить задачи на подзадачи.»  
   - <40% — «Много незавершённого. Расставь приоритеты.»  
   Если задач нет — «Пока нет задач. Добавь первую цель на день.»

3. **Глубокий анализ**  
   Заголовок «Глубокий анализ», кнопка «Обновить» (при пустом списке задач — неактивна). Запрос к бэкенду `/llm/plan`; ответ выводится текстом. Плейсхолдер: «Нажми «Обновить» для получения рекомендаций от AI».

---

### 3.6. Экран «Здоровье» — `WellnessView.swift`

- **Заголовок:** Здоровье  
- **Фон:** JarvisTheme.background  

**Блоки:**

1. **Советы**  
   Список рекомендаций от `aiManager.generateAdvice(from: store.tasks)` (время суток + количество задач за день). Если советов нет — «Добавь задачи для персональных рекомендаций».

2. **Питание**  
   Заголовок и счётчик калорий за сегодня. Поля «Блюдо» и «ккал», кнопка «+». На iOS — выбор фото (PhotosPicker); после выбора фото отправляется на бэкенд `/analyze-meal`, результат подставляется в блюдо/ккал и можно добавить запись. Список последних 5 приёмов пищи: название, дата/время, калории.

3. **Сон**  
   DatePicker «Лёг», DatePicker «Встал», кнопка «Сохранить ночь». Отображение последнего сна: «Последний сон: X.X ч».

4. **Активность**  
   Поля «Тип активности» и «мин», кнопка «+». Список последних 5 активностей: название, дата/время, минуты.

## 5. Модели данных и хранение

### 5.1. Задачи

- **PlannerTask:** id, title, notes, date, isCompleted, hasAlarm.  
- **PlannerStore:** массив задач, сортировка по date, сохранение в UserDefaults (`Config.Storage.tasksKey`).

### 5.2. Здоровье

- **MealEntry:** id, title, calories, date.  
- **SleepEntry:** id, start, end (часы через вычисляемое свойство).  
- **ActivityEntry:** id, title, minutes, date.  
- **WellnessStore:** meals, sleep, activities; сохранение снимка в UserDefaults (`Config.Storage.wellnessKey`).

### 5.3. Календарь / почта (UI)

- **CalendarEventItem:** id, title, notes, startDate.  
- **MailService.MessageDTO:** id, subject, from, date, snippet.

## 6. Backend API

**Базовый URL (клиент):** `http://jarvis-app.mooo.com:8000`

### 6.1. Реализованные эндпоинты (`jarvis-backend/main.py`)

| Метод | Путь | Назначение |
|-------|------|------------|
| GET | /health | Ответ `{"status": "ok"}` |
| GET | /calendar/events | Список событий (mock: 2 фиксированных события) |
| POST | /analyze-meal | Тело — бинарное изображение; ответ `{title, calories}` (эвристика по размеру в KB) |
| POST | /llm/plan | Тело `{tasks: [{title, notes, date, isCompleted}]}`; ответ `{advice}` (правила по доле выполненных и количеству задач) |

### 6.2. Ожидаемые клиентом, но отсутствующие на бэкенде

| Метод | Путь | Использование в приложении |
|-------|------|----------------------------|
| GET | /auth/status | Проверка авторизации (CalendarView, MailView) |
| GET (открытие в браузере) | /auth/google | Старт OAuth (кнопка «Войти через Google») |
| GET | /mail/messages?max_results=N | Загрузка писем в MailView |

В проекте есть `jarvis-backend/credentials.json` с Google OAuth (redirect_uri: `http://jarvis-app.mooo.com:8000/auth/callback`), но маршруты `/auth/status`, `/auth/google`, `/auth/callback` и `/mail/messages` в `main.py` не реализованы.

## 8. Вопросы к заказчику / команде

1. **Google Auth и почта**  
   Нужно ли в ближайшей версии реализовать на бэкенде `/auth/status`, `/auth/google`, `/auth/callback` и `/mail/messages` (с использованием имеющегося `credentials.json`), или Календарь/Почта пока остаются в режиме «ожидание бэкенда»?

2. **Источник событий календаря**  
   Оставить mock в бэкенде или планируется подключать реальный Google Calendar API после появления auth?

3. **Анализ фото блюда**  
   Оставить эвристику по размеру или планируется замена на ML/Vision API (например, для распознавания блюда и калорий)?

4. **LLM для «глубокого анализа»**  
   Нужна ли замена текущих правил на вызов внешнего LLM (OpenAI и т.п.) для генерации советов?

5. **Удаление задач**  
   Нужен ли в Плане сценарий удаления (swipe-to-delete или кнопка в RescheduleSheet)?

6. **Выбор AI-модели в UI**  
   Сейчас выбранная модель сохраняется, но в описанных экранах нет явного переключателя (Picker). Нужно ли добавить экран/блок настроек с выбором модели?
