## 1. ОБЗОР ПРОДУКТА

### 1.1. Что это
**Jarvis** — персональный AI-ассистент для планирования дня, управления задачами, календарём, почтой и здоровьем. Работает на всех Apple устройствах с полной синхронизацией.

### 1.2. Ключевые принципы
- **Offline-first:** Базовый функционал работает без интернета
- **Sync everywhere:** Данные синхронизируются между всеми устройствами
- **AI-powered:** Нейросеть помогает во всех аспектах (с возможностью отключения)
- **One codebase:** Единый код для iOS, iPadOS, macOS, watchOS

### 3.3. Модели данных

#### 3.3.1. Задача (Task)

```swift
struct JarvisTask: Identifiable, Codable {
    let id: UUID
    var title: String
    var notes: String
    var date: Date?                    // nil = Inbox
    var durationMinutes: Int
    var isAllDay: Bool
    var isCompleted: Bool
    var completedAt: Date?
    
    // Организация
    var categoryID: UUID?
    var priority: Priority             // low, medium, high, urgent
    var tags: [String]
    
    // Повторение
    var recurrence: RecurrenceRule?
    var completedRecurrenceDates: [Date]
    
    // Напоминания
    var reminders: [Reminder]          // Несколько напоминаний
    
    // Визуал
    var colorIndex: Int
    var icon: String
    
    // Мета
    var createdAt: Date
    var modifiedAt: Date
    var source: TaskSource             // manual, calendar, ai
    var linkedCalendarEventID: String? // Связь с Google Calendar
}

enum Priority: Int, Codable, CaseIterable {
    case low = 0, medium = 1, high = 2, urgent = 3
}

enum TaskSource: String, Codable {
    case manual, calendar, voice, ai
}

struct Reminder: Codable, Identifiable {
    let id: UUID
    var offsetMinutes: Int             // -30 = за 30 мин до
    var isEnabled: Bool
}

struct RecurrenceRule: Codable {
    var frequency: Frequency           // daily, weekly, monthly, yearly
    var interval: Int                  // каждые N дней/недель
    var daysOfWeek: [Int]?            // для weekly: [1,3,5] = Пн,Ср,Пт
    var endDate: Date?
    var maxOccurrences: Int?
}
```

## 4. ФУНКЦИОНАЛЬНЫЕ МОДУЛИ

### 4.1. ЗАДАЧИ (Plan)

#### 4.1.1. Inbox (Входящие)
- Задачи без даты — "мысли на потом"
- Быстрое добавление без указания времени
- Drag & drop на timeline для планирования

#### 4.1.2. Timeline (Расписание дня)
- Вертикальная временная шкала
- Задачи как блоки с цветом и иконкой
- Текущее время — красная линия
- Свободные слоты видны визуально

#### 4.1.3. Календарь (Неделя/Месяц)
- Переключение между видами
- Точки под датами — индикаторы задач
- Тап на день — переход к timeline этого дня

#### 4.1.4. Действия с задачами

| Действие | Как | Где |
|----------|-----|-----|
| Создать | Кнопка "+", голос, AI | Везде |
| Редактировать | Тап на задачу | Timeline, список |
| Завершить | Чекбокс | Timeline, список |
| Удалить | Свайп влево, контекстное меню | Список |
| Перенести | Drag & drop, редактирование | Timeline |
| Дублировать | Контекстное меню | Список |

#### 4.1.5. Умные функции
- **AI разбор:** "Позвонить маме завтра в 15:00" → задача с датой
- **Предложения времени:** AI предлагает слот на основе календаря
- **Автоперенос:** Незавершённые задачи переносятся на следующий день (опционально)

### 4.2. GOOGLE CALENDAR

#### 4.2.1. Функционал

| Функция | Описание |
|---------|----------|
| **Просмотр событий** | Все календари Google отображаются в приложении |
| **Создание событий** | Из приложения в Google Calendar |
| **Редактирование** | Изменение времени, названия |
| **Удаление** | С подтверждением |
| **Синхронизация** | Push-уведомления или polling каждые 2 мин |

#### 4.2.2. Интеграция с задачами
- События календаря отображаются на timeline (другим цветом/стилем)
- Можно конвертировать событие в задачу (и наоборот)
- Конфликты времени подсвечиваются

#### 4.2.3. Авторизация
1. Пользователь нажимает "Войти через Google"
2. Открывается Safari/ASWebAuthenticationSession
3. После входа — редирект на backend `/auth/callback`
4. Backend сохраняет токены, возвращает session
5. Приложение сохраняет session и использует для запросов

## 5. AI СИСТЕМА

### 5.1. Режимы работы

| Режим | Когда | Возможности |
|-------|-------|-------------|
| **Gemini (онлайн)** | Есть интернет | Полный AI: разбор, советы, фото, генерация |
| **Офлайн** | Нет интернета или выбрано | Базовый разбор по правилам |

### 5.2. Переключение
- В настройках: выбор модели по умолчанию
- Автоматический fallback на офлайн при отсутствии сети
- Индикатор текущего режима в UI

### 5.3. Сценарии использования AI

| Сценарий | Вход | Выход |
|----------|------|-------|
| **Разбор задачи** | "Позвонить Ивану завтра в 3" | `{title: "Позвонить Ивану", date: завтра 15:00}` |
| **Фото еды** | Изображение | `{title: "Паста карбонара", calories: 450}` |
| **Совет по дню** | Список задач | "Начните с важной встречи в 10:00..." |
| **Резюме письма** | Текст письма | Краткое содержание в 2-3 предложения |
| **Ответ на письмо** | Письмо + инструкция | Черновик ответа |

## 7. BACKEND API

### 7.1. Эндпоинты

#### Авторизация

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/auth/google` | Начать OAuth |
| GET | `/auth/callback` | Callback после Google |
| GET | `/auth/status` | Проверить авторизацию |
| POST | `/auth/logout` | Выйти |

#### Календарь

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/calendar/events` | Список событий |
| POST | `/calendar/events` | Создать событие |
| PUT | `/calendar/events/{id}` | Обновить |
| DELETE | `/calendar/events/{id}` | Удалить |

#### Почта

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/mail/messages` | Список писем |
| GET | `/mail/messages/{id}` | Одно письмо |
| POST | `/mail/messages/send` | Отправить |
| DELETE | `/mail/messages/{id}` | Удалить |
| POST | `/mail/messages/{id}/read` | Пометить прочитанным |

#### AI

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/ai/parse-task` | Разбор текста в задачу |
| POST | `/ai/analyze-meal` | Фото → калории |
| POST | `/ai/advice` | Совет по задачам |
| POST | `/ai/summarize` | Резюме текста |
| POST | `/ai/generate-reply` | Генерация ответа |

#### Синхронизация (опционально)

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/sync/tasks` | Синхронизация задач |
| GET | `/sync/tasks` | Получить задачи |

### 7.2. Безопасность
- HTTPS обязательно для production
- Токены хранятся только на backend
- Session ID передаётся в заголовке
- Rate limiting на AI эндпоинты
